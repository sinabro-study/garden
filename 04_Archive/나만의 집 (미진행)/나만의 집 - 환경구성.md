
### minikube를 활용하여 Local 환경 구성
```bash
brew install minikube

# Minikube Docker 환경 활성화
eval $(minikube docker-env)
docker info | grep Name

# docker hub가 아닌 local에서만 관리
./gradlew jibDockerBuild

minikube start --driver=docker --cpus=2 --memory=4096

# 각 환경 실행
kubectl apply -f deployments/db-pvc.yaml
kubectl apply -f deployments/db-deployment.yaml
kubectl apply -f deployments/db-service.yaml
kubectl apply -f deployments/app-deployment.yaml
kubectl apply -f deployments/app-hpa.yaml
kubectl apply -f deployments/app-service.yaml

# 파일 변경 시
kubectl diff -f deployments/  
kubectl apply -f deployments/

# 재배포 시
kubectl rollout restart deployment myhome-deployment

minikube image build -t myhome:latest .

# 각 환경 확인
kubectl get pvc
kubectl get pods
kubectl get svc
kubectl get hpa
```

```bash
# Deployment 제거 (Pod 전체 제거)
kubectl delete deployment myhome-deployment

# Docker Image Test
kubectl run myhome-test --image=myhome:latest --restart=Never --rm -it --image-pull-policy=IfNotPresent -- bash

# MySQL 접속
kubectl exec -it $(kubectl get pods -l app=myhome-mysql -o jsonpath="{.items[0].metadata.name}") -- bash

# App Log 확인
kubectl logs -f $(kubectl get pods -l app=myhome -o jsonpath="{.items[0].metadata.name}")

# Pod 확인
kubectl describe pods $(kubectl get pods -l app=myhome -o jsonpath="{.items[0].metadata.name}")
```

### 성능 지표를 확인하기 위해 Monitoring 세팅

- 간단한 지표 확인

```bash
kubectl get deployment metrics-server -n kube-system

# Metrics Server가 kubelet 인증서나 TLS 설정 문제로 kubelet 데이터를 갖오지 못함
# 기본 설치 상태에서는 minikube 클러스터 내부 IP 접근이 제한되어 probe 실패
minikube addons enable metrics-server

kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

kubectl get pods -n kube-system | grep metrics-server
kubectl describe pod -n kube-system metrics-server-5778bb4788-ngv9q
```
