## JWT (Json Web Token)
JWT는 JSON 객체를 사용하여 가볍게 자가수용적인 (Self-contained) 방식으로 정보를 안전하게 전달하기 위한 표준 (RFC 7519)입니다.

### 구조
- Header: 토큰의 타입과 사용 경우인 알고리즘(ex. HS256, RS256) 명시
- Payload: 토큰에 담을 정보(Claim) 세트
- Signature: 헤더와 페이로드를 비밀키로 해싱하여 생성한 서명

### 특징 및 단점
- 대중성: 거의 모든 언어와 프레임워크에서 라이브러리를 지원
- 유연성: 다양한 암호화 알고리즘 선택 가능
- 보안 취약점 (Critial)
	- **알고리즘 혼동 공격**: alg:none 으로 설정하거나 비대칭키를 대칭키로 오인하게 하여 검증 우회 가능
	- **데이터 노출**: 기본적으로 Base64 인코딩만 되어 있어 페이로드 내 민감 정보 쉽게 노출
