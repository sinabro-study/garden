### Reviewee?
: 코드를 작성하고 코드 리뷰를 신청한 동료

### Check List
- Coding Convention
	- [Google Java Convention](https://google.github.io/styleguide/javaguide.html)
	- [Kotlin Convention](https://kotlinlang.org/docs/coding-conventions.html)
- 기능 동작: 기능이 올바로 수행하는지 확인
- 테스트 코드: 통합/단위 테스트를 수행할 수 있는지 확인
- PR 내용 작성
	- 변경 내용을 요약하여 [[Reviewer]]가 전체 맥락을 파악할 수 있도록 해주세요.
	- 실제 테스트 결과를 작성하여 Reviewer가 확인할 수 있도록 해주세요.

### Mindset
- 코드와 나를 동일시 하지 않아야합니다.
    - 피드백을 받는 주체는 “내”가 아닌 **==“코드”==입니다.
- 리뷰를 반영하지 않아도 괜찮습니다.
    - Reviewer는 본인의 도메인을 전체적으로 이해하기 어려워 잘못된 리뷰를 남길 가능성이 있습니다.
    - 리뷰의 내용을 확인하여 의견을 주고 받는 과정이 필요합니다.
- Reviewer에게 모든 책임을 전가하지 않습니다.
    - 변경의 대한 책임은 리뷰이/리뷰어에게 있습니다.
- Reviewer가 변경 사항을 파악하기 쉽게 설명을 남겨주세요.
    - Reviewer는 세세한 내용까지는 알기 어렵습니다.  
        전체 맥락을 파악할 수 있도록 내용을 작성해주세요.
