## Code Review란?
- 본인을 제외한 다른 개발자들이 코드의 내요을 점검하고 피드백을 주는 과정

## Terms
- 리뷰어 ([[Reviewer]]): 코드 리뷰를 요청받은 개발자
- 리뷰이 ([[Reviewee]]): 코드 리뷰를 요청한 개발자

## Code Review의 목적은?
- 기능 정상 동작 확인
- 버그 가능성 발견
- 소프트웨어 품질 향상
	- 확장 가능한 코드
	- 가독성 있는 코드
- Domain 지식 공유
- 리뷰어, 리뷰이 서로 피드백을 받는 효과
- 개인이 아닌 팀이 제품을 만드는 효과

## Code Review 요청 방식은?
- GitHub [[Pull Request]] 활용

### 1단계
: Code Review를 도입하는 프로세스

- 브랜치 생성
- 요구사항에 맞는 코드 추가
- PR(Pull Request) 생성
- Code 변경의 맥락을 파악할 수 있도록 내용 추가

### 2단계  
: 작은 단위로 작업을 나누는 프로세스

- 브랜치 생성
- 요구사항에 맞는 코드 추가
	- 단, 변경 Line 수를 300~500줄 이내로 제한
-  PR (Pull Request) 생성
- Code 변경의 맥락을 파악할 수 있도록 내용 추가

### 3단계
: 규격화된 내용의 리뷰를 자동화하는 프로세스

- 브랜치 생성
- 요구사항에 맞는 코드 추가
	- 단, 변경 Line 수를 300~500줄 이내로 제한
-  PR (Pull Request) 생성
- Code 변경의 맥락을 파악할 수 있도록 내용 추가
- 필요한 체크 자동 확인 (Test, Convention, Size 등)
